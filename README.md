Набор скриптов для подготовки и разметки датасета **«вакансия — резюме»** для экспериментов по сопоставлению кандидатов и ранжированию в стиле ATS-систем.

## Что делает проект

Проект помогает:

- парсить сырые вакансии и резюме в единый табличный формат;
- нормализовать поля: должность, навыки, опыт, домен, seniority;
- формировать шаблон для экспертной разметки;
- собирать более полезную выборку пар для разметки:
  - сильные совпадения;
  - пограничные случаи;
  - вероятно нерелевантные пары;
- подготавливать данные для экспериментов по ранжированию кандидатов.

## Основные скрипты

### `parser_hr_dataset_v3_fixed.py`
Парсит сырые вакансии и резюме и сохраняет:

- `processed/vacancies.csv`
- `processed/resumes.csv`
- `annotations/labels_template.csv`
- `reports/parse_log.csv`

### `build_smart_labels.py`
Формирует более полезный шаблон разметки:

- `annotations/labels_template_smart.csv`
- `reports/pair_selection_report.csv`

## Быстрый старт

### 1. Создать и активировать виртуальное окружение

Windows:
```bash
py -m venv .venv
.\.venv\Scripts\activate
```

### 2. Установить зависимости

```bash
py -m pip install pandas python-docx openpyxl
```

### 3. Распарсить исходные файлы

```bash
py parser_hr_dataset_v3_fixed.py
```

После запуска будут созданы нормализованные таблицы вакансий и резюме.

### 4. Построить «умный» шаблон разметки

```bash
py build_smart_labels.py
```

Скрипт выберет сбалансированный набор пар для каждой вакансии:
- сильные совпадения;
- пограничные случаи;
- вероятно отрицательные примеры.

## Разметка

Для экспертной оценки используется шкала:

- `2` — релевантен;
- `1` — частично релевантен;
- `0` — нерелевантен.

Основные поля разметки:
- `annotator_1_label`
- `annotator_2_label`
- `final_label`
- `comment`

## Назначение выходных файлов

### `processed/vacancies.csv`
Нормализованные вакансии.

### `processed/resumes.csv`
Нормализованные резюме.

### `annotations/labels_template.csv`
Базовый шаблон разметки.

### `annotations/labels_template_smart.csv`
Шаблон разметки с более полезным отбором пар.

### `reports/parse_log.csv`
Лог парсинга: что обработано успешно, а что нет.

### `reports/pair_selection_report.csv`
Служебный отчёт по автоматическому отбору пар для разметки.

## Замечания

- Реальные вакансии и резюме могут содержать персональные данные.
- По этой причине в `.gitignore` исключены папки с сырыми и сгенерированными данными.
- В репозиторий лучше коммитить только код, шаблоны, конфиги и документацию.
- Если нужно показать пример данных публично, лучше добавить отдельную папку `examples/` только с анонимизированными примерами.
