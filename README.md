# ats-candidate-matching-experiment

Архивный проект по проверке гипотезы о «слепом пятне» в алгоритмическом первичном отборе кандидатов.

## Статус

Проект завершён и переведён в архив.  
Все рабочие файлы, папки, скрипты, данные и промежуточные результаты перенесены в `legacy/`.  
Новых экспериментов по этому проекту больше не будет.

## Исходная гипотеза

Центральным исследовательским предположением проекта был следующий тезис: **цифровая трансформация рекрутинга и алгоритмическое опосредование первичной коммуникации приводят к возникновению эффекта «слепого пятна»**, то есть к систематическому исключению компетентных соискателей из дальнейшего отбора вследствие того, что параметры их резюме не совпадают с критериями, используемыми автоматизированной системой.

Иными словами предполагалась ситуация при которой квалифицированный кандидат исключается из воронки найма до рассмотрения человеком не из-за недостатка компетенций, а из-за особенностей алгоритмической обработки резюме. Исходная гипотеза состояла в том, что при машинной интерпретации опыта может возникать расхождение между реальным профессиональным профилем кандидата и его алгоритмически распознанной версией, вследствие чего содержательно эквивалентные резюме могут получать разные оценки.

## Что было сделано

Для проверки этой гипотезы был собран базовый набор данных, включавший:
- **5 вакансий**;
- **65 резюме**;
- **100 пар** `вакансия — резюме` для ручной разметки.

Отобранные пары были распределены по трём группам:
- `top` — **40**;
- `borderline` — **30**;
- `random_negative` — **30**.

На этом наборе проверялось, насколько алгоритм в стандартной постановке отделяет сильные, пограничные и слабые совпадения.

Дополнительно была проведена отдельная экспериментальная попытка проверить, меняется ли результат отбора при изменении формы представления одного и того же опыта в резюме. Для этого был собран небольшой контролируемый набор:
- **3 кандидата**;
- по **4 версии резюме** на каждого;
- всего **12 экспериментальных резюме**.

Использовались версии:
- `original`;
- `ats_optimized`;
- `ai_assisted`;
- `weakly_structured`.

После прогона экспериментального генератора было получено:
- **15 пар** `вакансия — резюме`;
- по **3 пары на вакансию**.

Эта часть носила вспомогательный характер и использовалась как дополнительная проверка исходной гипотезы.

## Результаты

По итогам проекта исходная гипотеза **не получила подтверждения**.

Полученные результаты не дают оснований утверждать, что цифровая трансформация рекрутинга и алгоритмическое опосредование первичной коммуникации сами по себе приводят к возникновению эффекта «слепого пятна» в виде систематического исключения компетентных кандидатов из дальнейшего отбора.

Также не было получено существенных оснований считать, что в рассматриваемом pipeline ошибка накапливается до уровня устойчивого систематического сбоя отбора. Напротив, в рамках своей задачи алгоритм в целом показал нормальную и согласованную работу: использовал формализованные признаки, сопоставлял их с требованиями вакансии и ранжировал кандидатов в соответствии с заданной логикой.

## Итог

В рамках проекта не было получено подтверждения центрального тезиса о том, что цифровая трансформация рекрутинга и алгоритмическое опосредование первичной коммуникации закономерно приводят к возникновению эффекта «слепого пятна».

По полученным данным нельзя утверждать, что алгоритмический первичный отбор систематически исключает компетентных кандидатов именно из-за формы резюме.

Именно поэтому проект был остановлен и сохранён в архивном виде.
